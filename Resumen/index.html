Clase 1

Conceptos básicos de HTML

Ejemplo de cómo crear el código en HTML
<!DOCTYPE html>
<html>
<head>
<title>Título de la página</title>
</head>
<body>
<h1>Título principal</h1>
<p>Parrafo</p>
</body>
</html>

Se usó el editor (IDE) Visual Studio Code que facilita la codificación por los diferentes atajos que tiene, Recordar que la primera clase fue introductoria, se mostró conceptos básicos de HTML, hicimos el primer sitio que sorprendentemente era un "Hola mundo" y un segundo con la descripción de las etiquetas "head", "body", "h1", "h2" y "p" y otras más. Además de etiquetas, tambien encontramos atributos

Otra cosa importante que vimos es que las etiquetas tienen el formato de apertura con simbolos "<texto>" y el cierre con los simbolos "<texto/>"
Por ejemplo:

  •   <h1>Título principal </ h1>
  •   <h2>Título secundario </ h2>
  •   <p>Párrafo </ p>
      • ...
  • Existe excepciones, p.ej. la etiqueta de salto de línea no está codificada como un par de etiquetas de apertura y cierre. Es un elemento independiente o vacío, y está codificado como   < br>

Los comentarios en HTML dentro del editor se definen de la siguiente manera:

    <!-- todo lo que esté acá es un, puede estar en una linea o varias lineas-->

Ejemplos de Enlaces <a href="url (página web, imagen, archivo, lugar de la página)">Texto que va a aparecer en el enlace</a>

Enlaces en la misma pagina:

<a name="identificador de la etiqueta" >Texto al que se va a apuntar</a>

...

<a href="identificador de la etiqueta al que se quiere ir" >Texto que haga referencia al enlace</a>

Existen varios tipos de enlaces:

    La etiqueta <a> tiene atributos importantes ref= donde va el url y otro es el target= que por lo general será "-blank" para navegar al destino desde una pestaña nueva.

  • Enlaces absolutos
    • Son aquellos que tienen la dirección completa de la url, incluyendo desde el https://...
  • Enlaces relativos
    • Son las que enlazan a las paginas webs internas. Si tuviesemos carpetas diferentes, los metemos ahí. Los articulos por lo general se meten en carpetas
  • Enlaces locales
  • Enlaces a correo
  • Enlaces a archivos
  • Enlaces con imágenes

Los ejemplos de los enlaces se pueden ver en la web de la pag 2. Lo importante es recordar que se pueden hacer referencias dentro de la misma pagina, primero definiendo
<a name="Nombre de la etiqueta para la referencia">"todo lo que se escriba acá será el lugar al que se va a apuntar"</a>
Luego se puede hacer el llamado con la ref
<a href="Nombre de la etiqueta para la referencia">"Texto que se va a mostrar en la pantalla con el enlace"</a>

Clase 5

Introducción a CSS

Algo importante para tener en cuenta cuando trabajamos con CSS, es que los estilos que se definen en linea tienen prioridad antes que los internos y a su vez, los internos tienen prioridad ante los estilos externos

Una recomendación es que se use siempre estilos externos y que se use un archivo CSS con las cosas generales, evitar duplicar archivos CSS que tengan el mismo contenido.

La herencia, es importante saberlo.

Selectores:

  • Selector universal son los que afectan a toda la página.
  • Selector de etiquetas son los que se usan para etiquetas, tipo "h1" "section" "p".
  • Selector de clases se inician en el css con un punto "." y en el archivo html va dentro de una etiqueta con la palabra "class="xxxx""
  • Selector de identificador (id) la idea de estos selectores es que se apliquen a ciertos elementos en particular

El orden en el que se coloquen los selectores influye. En el CSS el último que aparezca reemplazará a los que están arriba. Por eso es importante el orden del CSS.

Atributos globales

Se pueden usar en todos los elementos HTML

  • style="estilo CSS" se usa para darle un estilo como si estuviera dentro del archivo CSS pero directamente en el archivo HTML
  • class="nombre" es el nombre con el que se definió una clase.
  • id="nombre" define el nombre del identificador que se usó
  • title="nombre"información adicional
  • hidden: evita que el elemento y todos los descendientes se muestren, puede usarse para apagar algunas cosas. Me imagino que se usará más con JavaScript

Especificidad

Es la prioridad que tienen los estilos, hay algunos que prevalecen antes que otros y se representan en el IDE cuando se pone el cursor encima y te muestra el nivel o el peso del selector.

La jerarquía de la especificidada es la siguiente:

  • (1,0,0,0,0) !important: se coloca "!important" despues de cada estilo y se usa con cualquier selector.
  • (0,1,0,0,0) inline: se coloca dentro de la etiqueta "style="color:red" y toma el estilo toda la etiqueta.
  • (0,0,1,0,0) ID: se coloca con la etiqueta "#nombre {color:red}"
  • (0,0,0,1,0) Clases, atributos y pseudoclases: se coloca con " .nombre{color:red}"
  • (0,0,0,0,1) Etiquetas y pseudoelementos: se coloca con la etiqueta " nombre-de-etiqueta {color:red}"

Unidades de medidas

La unidad de medida es el pixel (el minimo cuadradito de luz del monitor)

En css podemos diferenciar los parrafos definidos con la definición de clases. Por ejemplo

  • Unidades de medidas absolutas

    • Por lo general es el "px" pixel la más común. Pero tambien se puede usar "cm", "mm", pt (puntos), in pulgadas

  • Unidades de medidas relativas
    • em es la unidad de la M estandar a 16px, por lo que todo es múltiplo de ella. Para pasar de em a px se hará una regla de 3
    • rem es una unidad que significa root em. Primero se define la raíz que por defecto es 16px. Si cambiamos el font-size, por defecto cambia el tamaño de la fuente porque la base ha cambiado.
    • porcentaje(%) es en funcion del tamaño base. Es parecido al REM pero en porcentaje
  • Unidades flexibles
    • vw Se basa en el ancho de la pantalla. Por ejemplo en una pantalla de 1200px de ancho, 1vw es 12px
    • vh Se basa en el alto de la pantalla. Por ejemplo en una pantalla de 800px de alto, 1vh es 8px
